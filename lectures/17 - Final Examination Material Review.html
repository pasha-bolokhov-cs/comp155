REVIEW TOPICS

What you have been doing in the labs since the midterm builds on the material you covered prior to the midterm. As a result, the final will be comprehensive, but with a heavier weight toward material covered since the midterm.

Pre-midterm (but still included):

  • Chapters 1 to 6 - (those parts found in the online notes) and normalization. Go over all your labs.
  • The notes covered the early part of chapter 7 on UNION and INTERSECT but that won't be on the midterm.
  • Do not memorize all the functions - but know how to use the ones from the labs: ROUND, TRUNC, MONTHS_BETWEEN, ADD_MONTHS, TO_DATE, TO_CHAR, UPPER, LOWER, SUM, COUNT, AVG, MAX, MIN, SOUNDEX, NVL, LTRIM, LENGTH, INITCAP, CONCAT.
  • Recognize single row functions and group functions.
  • Be able to use the TO_CHAR() formatting options.
  • Know the basic Oracle data types on page 12 of the book.
  • Know the basic form of the SELECT statement.
  • Know column aliases.
  • Know how to use the LIKE operator with wildcards % and _
  • Know when to use the * wildcard in a SELECT
  • Know how to use the IN and BETWEEN operators
  • Know when to use NULL in a select statement.
  • Know when to use || and literals ' ' in a select.
  • Know the difference between the use of single quotes and double quotes.
  • Know how to sort ascending and descending.
  • Know how to use table aliases.
  • Know how to calculate a Cartesian product.
  • Know how to recognize when a Cartesian product may be created.
  • Know how to do SQL99 and Oracle format inner and outer (left and right) joins
  • Know the difference between using USING and ON in a SQL99 join.
  • Know how to group by columns
  • Know how to filter groups of rows using HAVING
  • Know the purpose of SYSDATE
  • Know the default date format Oracle uses
  • Know the COLUMN HEADING FORMAT syntax for the SQL*Plus
  • Know the order of precedence for logical operators AND and OR.
  • Know the comparison operators.
  • Know what SET ECHO OFF/ON  and SPOOL does in SQL*Plus
  • Know what SET PAGESIZE 50 and SET LINESIZE 100 do in SQL*Plus
  • Know the purpose of the SQL*Plus DESCRIBE command
  • Know in which order subqueries are evaluated.
  • For subqueries know when to use =, >, <, <> and IN, =ANY, NOT IN, etc for multiple row subqueries.
  • Know how to normalize to third normal form.
  • Be able to optimize after you have completed your normalization.
  • Be familiar with terminology such as: primary key, foreign key, candidate key, alternate key, composite key, surrogate key, functional dependency, partial functional dependency, transitive dependency, anomalies

Post-midterm:

  • Chapters to 7 - 11 (those parts found in the online notes) and normalization.
  • Be familiar with the use of the set operators (UNION, UNION ALL, INTERSECT, MINUS)
  • Be familiar with the use of the CASE expression.
  • Know how to add rows with the INSERT statement.
  • Know how to delete rows with a DELETE statement.
  • Know the difference between a DELETE and TRUNCATE statement.
  • Know how to modify one or many rows with the UPDATE statement
  • Be able to create a primary key, foreign key and check constraint.
  • Know how to grant privileges on a table/column to a user
  • Know how to revoke privileges from a user
  • Understand the purpose of setting privileges
  • Understand the difference between DML statements and DDL statements.
  • Be able to use COMMIT and ROLLBACK.
  • Know how to create, alter, rename, truncate and drop a table.
  • Know how to create, modify, and drop a sequence.
  • Know the purpose of a sequence and when to use one.
  • Be able to use a sequence in an INSERT or UPDATE statement.
  • Know the difference between the sequence object's pseudocolumns NEXTVAL and CURRVAL.
  • Know how to create, modify, and drop an index.
  • Know when you should use an index, and when an index is created automatically.
  • Know how to create, modify, and drop a view.
  • Know the purpose of using a view, and what operations can be performed on a view (e.g. inserting, updating and deleting). (Note; inline views will NOT be on the final.)
  • Know how to create and drop a synonym.
  • Know that a synonym can be used for views as well as tables.
  • Know how to create, replace and drop a trigger.
  • Know the different parts of a trigger.
  • Be able to write a basic trigger.
  • Know the difference between a row level and statement level trigger and be able to write one of each.
  • Know what happens to views, triggers, indexes, and synonyms when the associated table is dropped.
  • Know the purpose of JDBC. (Note: You will NOT be writing any Java on the final.)
  • Know basic database terminology (e.g. relations, tuples, attributes etc), and the purpose of Codd's 12 rules. You will NOT be asked to list any of the rules.
  • Understand the purpose of data dictionaries, and know how to use them to get information about your or someone else's schema.
  • Be aware that relational algebra underlies SQL.
  • Be aware of the elements making up the definition of Big Data.
