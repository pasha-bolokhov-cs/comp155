Objective

  • Create, alter, rename and drop tables
  • Create tables
  • Describe the datatypes that can be used when specifying column definitions
  • Alter table definitions
  • Drop, rename and truncate tables
  • Describe constraints
  • Create and maintain constraints.
  • Add comments to a table.

Do

  • Pages 300-306 (top half). Read the sections on creating a table, getting information on tables and columns, and altering tables
  • Pages 306-312. Constraints are so important that they will have a lab to themselves. The next lab will be on constraints.
  • Pages 313-315. Read the sections on renaming a table, adding comments plus truncating and dropping tables.
  • Page 314. By default in Oracle11g, tables are not simply dropped. They are made invalid so that you can retrieve them later if you dropped them by mistake. To completely drop a table without the option of later retrieval, you must use the command:
    DROP TABLE tablename PURGE;
    In order to make our installation of Oracle compatible with older versions and other DBMSs, the facility to retrieve dropped tables has been turned off. You may use a simple DROP TABLE tablename; statement to drop any of your tables.
  • Try the various examples in the chapter on creating tables and modifying them.

Assignment

Start spooling so that as you work through the following steps you will have a record of the commands and the results. Only properly spooled results will be accepted. Be sure to use proper formatting for all SQL statement (e.g. put clauses on separate lines).

  1. Create a table in your schema called MY_STUDENT with the following attributes:
    mystudent_id number(5)
    mylname varchar2(25)
    mystartdate date      ← Make the column default to the system date
  2. Use the describe command to show the structure of your new MY_STUDENT table.
  3. Modify the MY_STUDENT table to allow for longer last names (i.e. mylname). Increase its size to 50.
  4. Confirm your modification. (Hint: Use the SQL*Plus describe command.)
  5. Pretend you are a student being added to the table. Make up a student id and insert your name into the MY_STUDENT table. Allow the start date to default to the system date - in other words, do NOT explicitly give a date.
  6. List the contents of the table to show that your record has been entered correctly.
  7. Add a comment on the table which says "Lab 11 Table"
  8. Verify that this comment has been added to your table. (Only show the comments for your table.)
  9. Create a table called My_Student (note the use of upper and lower case) based on the structure of your MY_STUDENT table. Use the CREATE TABLE .. AS SELECT command as described in the attached note. Name the columns in your new table StudentNo, Last Name, BeginDate (note the use of upper and lower case).
  10. Show the structure of your new My_Student table. (Hint: Use the SQL*Plus describe command.)
  11. Confirm that both the MY_STUDENT and My_Student tables are stored in the data dictionary.  (Hint: Use the SELECT table_name FROM user_tables; command.)
  12. Drop the MY_STUDENT table; being sure to purge.
  13. Rename the My_Student table to MY_STUDENT.
  14. Confirm that only the MY_STUDENT table remains in your schema.
  15. Drop your MY_STUDENT table; being sure to purge.
  16. Verify that MY_STUDENT has been dropped.
  17. Spool off

Print and hand-in the results (if you do not want to upload the output file to your lab 11 dropbox on D2L).
